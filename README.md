# Data Entry, Cleaning & Validation Project

## Overview
This project demonstrates a complete data entry workflow, including raw data processing, data cleaning, validation, and error tracking.

It highlights real-world practices used in data entry, data quality assurance (QA), and data processing roles.

---

## Objectives

- Perform accurate data entry
- Identify and correct data errors
- Apply validation rules to ensure data quality
- Track and document errors during processing

## Dataset Description

The dataset includes:
- Name
- Age
- Email
- Phone number
- City

The raw dataset contains intentional errors to simulate real-world data entry scenarios.

---

## Data Cleaning Process

- Filled missing values (e.g., age, city)
- Corrected invalid email formats
- Standardized city names (e.g., Alex → Alexandria)
- Fixed phone number formats

---

## Data Validation Rules

- No missing values allowed
- Email must be valid (contains @ and .com)
- Phone number must be 11 digits and valid prefix
- Age must be within a logical range (18–60)
- City names must be standardized

---

## Error Tracking

All detected errors are documented in the error log, including:
- Type of error
- Original value
- Corrected value
- Action taken
